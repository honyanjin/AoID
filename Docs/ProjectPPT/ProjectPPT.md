# AoID (Association of Independent Developers) 발표 PPT 계획서

> **발표자**: 이호진 (1인 풀스택 개발자)
> **프로젝트명**: AoID - 독립 개발자 협회 플랫폼

---

### 타이틀 슬라이드

### 핵심 키워드
- **독립 개발자** - Independent Developer
- **커뮤니티** - 기술 공유와 협업

- **플랫폼** - 통합 서비스 제공

### 발표 오프닝 멘트

## 슬라이드 2: 서비스 기획배경

### 배경 설명
```
┌─────────────────────────────────────────────────────────────┐
│  💡 왜 AoID를 만들었는가?                                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 독립 개발자의 증가                                         │
│     • 프리랜서, 1인 개발자 급증                                 │
│     • 재택근무 확대로 독립 개발 문화 확산                        │
│                                                             │
│  2. 고립된 개발 환경의 문제                                     │
│     • 혼자 개발하며 겪는 기술적 어려움                           │
│     • 정보 공유 채널 부족                                      │
│     • 네트워킹 기회 제한                                       │
│                                                             │
│  3. 기존 플랫폼의 한계                                         │
│     • 단순 Q&A 또는 구인구직에 집중                             │
│     • 종합적인 커뮤니티 부재                                    │
│     • 한국 독립 개발자 특화 서비스 부족                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 시장 현황
| 구분 | 현황 |
|------|------|
| 국내 프리랜서 개발자 | 약 30만명 추정 |
| 연평균 성장률 | 15% 이상 |
| 주요 니즈 | 기술 공유, 협업, 네트워킹 |

### 기획 동기
- 독립 개발자로서 직접 느낀 고립감과 정보 부족
- 체계적인 협회 형태의 커뮤니티 필요성 인식
- 단순 커뮤니티를 넘어선 종합 플랫폼 구상

| 네트워킹 | Connection Round | 프로필 연결, 매칭 |

## 슬라이드 5: 서비스 시연 흐름

### 시연 시나리오 (10분)

#### Part 1: 회원가입 및 로그인 (2분)
```
1. 홈페이지 메인 화면 소개
2. 회원가입 화면 시연
   - 이메일/비밀번호 가입
   - Google OAuth 가입 시연
3. 로그인 후 메인 화면 변화 확인
```

#### Part 2: 게시판 기능 (3분)
```
1. 게시판 목록 페이지
   - 카테고리별 필터링
   - 검색 기능
2. 글 작성
   - 리치 텍스트 에디터 (React Quill)
   - 이미지/파일 첨부
3. 댓글 및 대댓글
4. 좋아요 기능
```

#### Part 3: DevShop & 결제 (3분)
```
1. DevShop 상품 목록
2. 장바구니 기능
3. DevCoins 충전 (토스페이먼츠 테스트)
4. 상품 구매 완료
```

#### Part 4: 관리자 기능 (2분)
```
1. 관리자 대시보드
   - 통계 그래프 (Recharts)
2. 회원 관리
3. 홈페이지 섹션 편집
   - Hero 섹션 수정
   - Motion Box 편집
```

### 시연 체크리스트
- [ ] 테스트 계정 준비 (일반/관리자)
- [ ] 토스페이먼츠 테스트 모드 확인
- [ ] DevCoins 잔액 초기화
- [ ] 샘플 게시글/상품 준비
- [ ] 네트워크 환경 확인

### 예상 질문 및 답변

#### Q1: 1인 개발에서 가장 어려운 점은?
> **A**: 모든 역할을 혼자 수행해야 하는 것입니다. 특히 코드 리뷰 없이 개발하면
> 품질 관리가 어렵습니다. 이를 보완하기 위해 TypeScript를 적극 활용하고,
> ESLint/Prettier로 코드 스타일을 통일했습니다.

#### Q2: 왜 이 기술 스택을 선택했는가?
> **A**: React는 컴포넌트 기반 개발로 재사용성이 높고, Express는 Node.js 생태계의
> 풍부한 라이브러리를 활용할 수 있습니다. PostgreSQL은 복잡한 관계형 데이터와
> 트랜잭션 처리에 강점이 있어 선택했습니다.

#### Q3: DevCoins 시스템의 활용 계획은?
> **A**: 현재는 상점 구매에 사용되지만, 향후 프로젝트 협업 보상, 기술 공유 인센티브,
> 이벤트 보상 등으로 확장할 계획입니다.

#### Q4: 결제 시스템 구현 시 주의점은?
> **A**: 테스트 환경과 실제 환경의 차이를 이해하고, 모든 실패 케이스에 대한
> 에러 처리가 필수입니다. 또한 결제 정보는 반드시 서버에서 검증해야 합니다.

#### Q5: 향후 확장 계획은?
> **A**: 모바일 앱 개발, 실시간 알림 기능, AI 기반 프로젝트 매칭 등을
> 계획하고 있습니다.
